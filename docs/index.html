Why this matters

EvidenceWeaver focuses on the part of agentic RL that still feels under-built.

Many long-horizon agents are optimized for task success while under-optimizing for evidence quality, citation faithfulness, and multi-hop reasoning structure.

Reward decomposition beats scalar outcome reward

Separate answer quality, support coverage, diversity, and efficiency so training pressure is visible instead of hidden behind one number.

Evidence should be structured

A research agent should maintain an evolving graph of claims, sources, contradictions, and unresolved gaps instead of dumping text into flat context.

Stability is a first-class problem

Long-horizon policy collapse is easy to miss. Diagnostics and honest failure cases belong in the artifact, not in the appendix.

System map

A compact loop that stays legible as the repo grows.

EvidenceWeaver is intentionally small: a baseline agent, an evidence graph, a reward bundle, an evaluator, and benchmark slices that can be read without private infra.

Agent

Plans, searches, reads, writes, and emits a typed run artifact.

Environment

Exposes reproducible search and snapshot-style evidence interactions.

Evidence graph

Stores sources, claims, support edges, duplicates, and open questions.

Reward server

Breaks scoring into answer, citation, chain, diversity, and efficiency terms.

Trainer

Reserved for the next stage: online or hybrid RL over the same interfaces.

Evaluator

Keeps offline judgment separate so benchmark claims stay honest.

Run tests

PYTHONPATH=src python3 -m unittest discover -s tests -v

Run the baseline

PYTHONPATH=src python3 -m evidenceweaver.agent.baseline \
  benchmarks/snapshot_v0/tasks/agent_training_stack_task.json \
  --output /tmp/evidenceweaver_run.json

Score a run

PYTHONPATH=src python3 -m evidenceweaver.eval.offline \
  benchmarks/snapshot_v0/tasks/agent_training_stack_task.json \
  /tmp/evidenceweaver_run.json \
  --emit-scored-run /tmp/evidenceweaver_scored_run.json

Open-source ready lanes

Contributions should sharpen the benchmark, not decorate around it.

EvidenceWeaver is designed to earn trust slowly. High-signal contributions improve reproducibility, failure visibility, benchmark design, or reward clarity.

benchmark:

Propose a task family, snapshot recipe, or evaluation setup that reveals a real gap.

reward:

Suggest a term, anti-hacking check, or scoring rule with explicit failure analysis.

failure-case:

Document a trajectory or policy behavior that broke in an interesting, reproducible way.

docs / infra / eval

Clarify the story, harden the artifact path, or make benchmark assumptions easier to inspect.
